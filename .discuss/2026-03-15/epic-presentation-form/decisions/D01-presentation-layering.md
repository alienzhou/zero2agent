# D01：课程呈现层级

## 状态
✅ Confirmed

## 决策

课程内容按四层结构呈现：

1. **README / 首页**
   - 负责一眼看懂项目定位与学习路径
   - 只放轻量总览，不展开 Story

2. **Roadmap 总览页**
   - 负责展示完整学习地图
   - 以 Epic 为主轴，呈现阶段目标、完成效果、当前状态与入口链接

3. **Epic 页**
   - 负责解释单个阶段的目标、必要性、包含的 Story，以及阶段完成后的收获
   - 保持为轻量课程导览页

4. **Story 页**
   - 负责承载具体课题
   - 作为学习者真正进入内容、阅读实现与扩展思考的主单元

其中：

- README 不承担完整 Roadmap 页职责
- Roadmap 总览页与 Epic 页之间保持“全图”与“阶段”的层级差异
- Story 级的设计取舍与扩展阅读不回挂到 Epic 页

## 理由

1. README 面向的读者最杂，需要保持轻量与入口感。
2. Roadmap 总览页适合承担“完整地图”职责，避免首页过重。
3. Epic 与 Story 需要明确分工，否则内容会在“阶段导览”和“具体课题”之间反复重叠。
