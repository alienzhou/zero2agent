# 课程呈现草稿

## 总体结构

当前建议采用四层呈现：

1. README / 首页
2. Roadmap 总览页
3. Epic 页
4. Story 页

它们之间的关系是：

- README：门口
- Roadmap：地图
- Epic：阶段导览
- Story：具体课题

## 各层建议展示粒度

### README / 首页

建议只展示：

- 项目定位
- 学习方式说明
- 5 个 Epic 的轻量卡片
- 当前推荐起点
- 前往完整 Roadmap 的入口

不建议：

- 在 README 中展开 Story 列表
- 把完整课程树直接塞进首页

### Roadmap 总览页

建议每个 Epic 展示：

- Epic 标题
- 一句话阶段目标
- 一句话完成效果
- 当前状态（planned / in progress / done）
- 跳转到 Epic 页的链接

可选：

- 列出 Story 标题，但不展开内容

### Epic 页

建议固定为四块：

1. 这个 Epic 要把我们带到哪里
2. 为什么现在做这个 Epic
3. 这个 Epic 包含哪些 Story
4. 完成这个 Epic 后，你会得到什么

其中 Story 建议仅保留：

- 标题
- 一句话摘要
- 入口链接

### Story 页

建议固定为四块：

1. 这次我们要解决的问题和目标
2. 这次实现的关键点
3. 做完后的效果
4. 扩展阅读

## 风格提醒

- README 要轻，不要承担完整课程文档职责
- Epic 要轻，不要写成项目方案
- Story 要实，但不要替代技术实现文档
- 扩展阅读要贴近具体课题，而不是漂浮在阶段层

## 目前最稳的角色分工

- README：帮助新读者判断“这是不是我想学的东西”
- Roadmap：帮助学习者建立全图感
- Epic：帮助学习者理解“这一阶段为什么存在”
- Story：帮助学习者理解“这一步在解决什么，以及如何观看实现”
