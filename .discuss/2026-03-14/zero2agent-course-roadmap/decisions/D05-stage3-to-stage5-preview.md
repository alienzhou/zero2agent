## Epic 3：基础能力与产品化

核心目标：

- 让 Agent 的使用体验更像一个真实产品，而不只是调试脚本
- 补齐基础交互展示、日志与产品外壳

当前涉及方向：

- 架构改造：让 Agent 作为核心运行调用，被 TUI / Web UI / 其他 UI 连接
- 先落一版 TUI，作为产品化的第一形态
- 会话管理：从单轮对话走向更真实的产品会话
- 功能补齐：如 Checkpoint 这类产品功能
- **安全边界 Story**：
  - Approval（审批模式）
  - 权限控制
  - 工作目录校验

## Epic 4：健壮性与上下文管理

核心目标：

- 解决复杂异常情况
- 管理超长上下文与工具上下文膨胀

当前涉及方向：

- 异常处理：确保 Agent 在复杂情况下仍能高效、正常运行
- 上下文管理：处理长上下文、工具上下文、超长输入等问题
