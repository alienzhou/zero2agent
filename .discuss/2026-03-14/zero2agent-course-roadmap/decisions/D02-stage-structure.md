# D02：Stage 总体结构与课程分层

## 状态
✅ Confirmed

## 决策

当前 Roadmap 的阶段结构收敛为：

- **Stage 1：能看 / 能查**
- **Stage 2：能动 / 能改 / 能执行**
- **Stage 3：基础能力与产品化**
- **Stage 4：健壮性与上下文管理**
- **Stage 5：扩展能力**

其中：

- 扩展能力暂时后置，不抢前半程主线。
- 浏览器能力暂时不进入 Stage 2 主线。
- 更深的上下文管理、错误反馈、异常处理，优先放到 Stage 4 这类“深水区”阶段。

## 备注

Stage 5 之后，未来还可能继续扩展：

- Agent Teams
- Plan
- Task List
- Long Running

这些先保留为远期方向，不在当前版本细化。

## 理由

1. 前半程先建立“看得懂 -> 动得了”的清晰成长路径。
2. 产品化、稳健性、扩展性都重要，但不应在前两个阶段同时展开。
3. 先把已有能力做深、做稳，再横向扩展，更符合课程节奏。
