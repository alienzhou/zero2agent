# Epic 1：能看 / 能查

> 让 Agent 从 0 开始，先学会安全地读取和定位信息，再建立对 ReAct 循环的基本直觉。

---

## 这个 Epic 要把我们带到哪里

在进入写文件、执行命令这些更强能力之前，我们先把 Agent 带到一个更稳的起点：

- 它能理解用户问题
- 它能通过循环调用模型与工具推进任务
- 它能安全地查看文件和目录
- 它能在工作区中逐步定位信息

这一阶段的目标，不是让 Agent 很强，而是让它先**可运行、可解释、可跟练**。

---

## 为什么现在做这个 Epic

如果一开始就把“写文件”“执行终端”“权限边界”全部压进来，学习者会同时面对太多变量，很难建立稳定直觉。

所以 Epic 1 要先把主线收敛到一个更安全的问题上：

- 先让 Agent 学会“看”
- 再让 Agent 学会“查”
- 最后回看 Prompt 和循环结构，把这套最小闭环讲清楚

这样做的好处是：

- 能快速跑通最小可用闭环
- 只读能力更安全，适合作为课程开场
- 后续进入写能力和执行能力时，学习者已经有了稳定参照物

## 完成这个 Epic 后，你会得到什么

完成 Epic 1 后，你会得到两类收获：

### Agent 能力上的收获

- 一个安全、可解释的最小只读 Coding Agent
- 对“模型思考 -> 工具调用 -> 工具结果回传 -> 继续循环”的基本认识
- 对只读工具、搜索工具和基础 Prompt 结构的直觉

### 学习上的收获

- 知道一个最小 Agent 应该先从什么地方起步
- 能用真实代码和 commit 去理解 ReAct 模式
- 为进入 Epic 2 的“能动 / 能改 / 能执行”做好铺垫
