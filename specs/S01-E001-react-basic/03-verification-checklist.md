# S01-E001: 验收检查清单

> 开发过程中需要特别关注的检查项。

---

## 功能验收

### 基础功能

| 检查项 | 优先级 | 模块 |
|--------|--------|------|
| 能正常调用 Anthropic API | P0 | LLM |
| 工具定义正确传递给 API | P0 | Tools |
| 能解析 `stop_reason: tool_use` | P0 | Loop |
| 工具执行结果正确返回给模型 | P0 | Loop |
| 循环能正常终止 | P0 | Loop |

### read_file 工具

| 检查项 | 优先级 | 模块 |
|--------|--------|------|
| 能读取存在的文件 | P0 | Tools |
| 文件不存在时返回错误信息 | P0 | Tools |
| 支持行号范围读取 | P1 | Tools |
| 输出带行号前缀 | P1 | Tools |

### list_directory 工具

| 检查项 | 优先级 | 模块 |
|--------|--------|------|
| 能列出目录内容 | P0 | Tools |
| 目录不存在时返回错误信息 | P0 | Tools |
| 支持 recursive 参数 | P1 | Tools |
| 递归时正确显示层级缩进 | P1 | Tools |
| 显示完整相对路径 | P1 | Tools |

---

## 边界场景

### 输入边界

| 检查项 | 优先级 | 说明 |
|--------|--------|------|
| 空路径处理 | P1 | `read_file({ path: "" })` |
| 特殊字符路径 | P2 | 文件名含空格、中文等 |
| 超大文件 | P2 | 可能导致 token 超限 |

### 状态边界

| 检查项 | 优先级 | 说明 |
|--------|--------|------|
| 无限循环防护 | P0 | 设置最大循环次数 |
| API 返回异常 | P1 | 网络错误、超时等 |
| 空响应处理 | P1 | content 为空数组 |

---

## 环境检查

| 检查项 | 优先级 | 说明 |
|--------|--------|------|
| 环境变量缺失提示 | P0 | ANTHROPIC_API_KEY 未设置时的友好提示 |
| Node.js 版本兼容 | P1 | 确保 Node 22+ |
| TypeScript 编译通过 | P0 | 无类型错误 |

---

## 已知限制

以下是当前版本的已知限制，需在文档中说明：

| 限制 | 说明 |
|------|------|
| 无路径安全检查 | 可以访问任意路径（后续迭代添加） |
| 只支持只读操作 | 不支持文件写入、删除等 |
| 无 token 计数 | 可能因大文件导致 token 超限 |
| 无流式输出 | 需等待完整响应 |

---

## 优先级定义

- **P0** = 必须检查（核心功能，不通过则不可发布）
- **P1** = 应该检查（重要功能，影响用户体验）
- **P2** = 建议检查（边缘场景，可后续完善）
